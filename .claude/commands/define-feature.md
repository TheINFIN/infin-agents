You are executing the **Feature Definition Workflow** - a multi-agent process for comprehensive feature definition.

## Workflow Stages

Execute each stage in sequence, clearly labeling your output:

### Stage 1: Problem Definition & Scoping (as Product Manager)
- Define the problem statement clearly
- Identify target users
- Set success metrics and KPIs
- Define scope (in/out)
- Document assumptions and risks
- Assess priority

### Stage 2: User Research & Insights (as User Researcher)
- Define user personas for this feature
- Map the user journey
- Document jobs-to-be-done
- Identify key user needs and pain points

### Stage 3: Experience Design (as UX Designer)
- Define user flows
- Create wireframes (ASCII or description)
- Specify interactions and states
- Note accessibility requirements
- Document edge cases

### Stage 4: Content & Messaging (as Copywriter)
- Write UI copy (labels, buttons, CTAs)
- Create error messages
- Write help/tooltip text
- Ensure consistent voice and tone

### Stage 5: User Story Refinement (as Story Groomer)
- Break into user stories
- Write acceptance criteria (Given/When/Then)
- Identify dependencies
- Estimate relative complexity

### Stage 6: Technical Assessment (as Architect)
- Assess technical feasibility
- Identify integration points
- Flag technical risks
- Suggest implementation approach

### Stage 7: Final Specification
Consolidate into a complete feature spec with:
- Executive summary
- Success metrics
- UX specification
- Content/copy
- User stories (ready for development)
- Technical notes

## Output Format

Structure your response with clear headers for each stage. Be thorough but actionable.
